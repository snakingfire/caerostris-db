### WHAT CHANGED IN v2 (the DA-1 / BC-4 fix) ###

  - Data-object identity re-keyed:
        v1:  ObjId(v, k)       == "v"+v+"-"+k                 (version,shard)
        v2:  ObjId(v, w, a, k) == "v"+v+"-w"+w+"-a"+a+"-k"+k  (version,writer,
                                                               ATTEMPT,shard)
    models the ADR content-addressed key `db/data/<content-hash>/<shard>.col`
    (or the attempt-scoped `db/data/v<V>/<epoch-or-uuid>/<shard>.col`). Two
    writers racing the SAME version now stage DISTINCT ids; the idempotent
    set-union collapse that hid the tear is gone.
  - Added `writerAttempt[w]` (monotone per-writer counter, bumped on every
    AcquireLease) — the per-write token; a retry after fencing stages fresh ids.
  - Added action `ZombieLateWrite(w)` — the DA-1 attack made REPRESENTABLE: a
    fenced/stalled writer re-issues its data PUT. With attempt-scoped keys it
    lands on the writer's OWN key (an orphan), never a referenced one.
  - Added invariants `OrphansNeverReferenced` and `NoOverwriteOfReferenced` —
    turn "write-once immutability" from an ASSERTION into checked properties.
  - Added non-vacuity probes `DistinctIdsProbe`, `ZombieWroteProbe`.
  - GC now subtracts surviving-manifest references (defence-in-depth
    ref-counted GC; dovetails with SPIKE-0003 / decision 0015).
  - SYMMETRY narrowed to readers only (writer ids now carry object-id payload;
    permuting writers would be unsound for the object-layer invariants).

### 1. SAFETY check (all invariants, with reader-symmetry) ###
$ java -cp tla2tools.jar tlc2.TLC -config commit_protocol.cfg -deadlock \
      -workers auto commit_protocol.tla
Invariants checked: TypeOK, NoTornCommit, SnapshotIsolation,
  AtMostOneCommitPerVersion, ManifestImmutable, GCSafety,
  OrphansNeverReferenced, NoOverwriteOfReferenced, LatestIsDurable
  (and their conjunction SafetyInvariant).

EXPECTED: No invariant violations. Hand-derivation of the two NEW invariants
(the others are unchanged from v1 and their arguments carry over because
attempt-scoped ids only make object sets MORE distinct, never less):

  OrphansNeverReferenced — for a fenced/crashed writer w whose target version
    is committed: the committed manifest's `objs` is exactly the WINNER's staged
    set StagedObjs(target, winner, a_win); w's set is StagedObjs(target, w, a_w).
    The ids embed the writer token, so winner # w => the sets are DISJOINT
    (left disjunct holds). The one case where w IS the committer-then-crashed
    (manifests[target].objs = writerObjs[w]) is admitted by the right disjunct.
    Either way the invariant holds.  QED

  NoOverwriteOfReferenced — for any committed manifest v and writer w, if
    manifests[v].objs ∩ writerObjs[w] # {} then the shared id embeds version v
    AND a single (writer,attempt) token; manifests[v].objs is precisely the
    winner's staged set, so w must BE that winner and writerObjs[w] =
    manifests[v].objs. No OTHER writer's staged set can intersect a referenced
    object set => a stale PUT (ZombieLateWrite) by a non-owner cannot collide
    with a referenced key. No last-write-wins tear of a live object exists.  QED

  ZombieLateWrite preserves all `subseteq dataObjects` invariants: it only ADDS
  ids to dataObjects (monotone), so any subset relation that held still holds.

  GCOldVersion (ref-counted) preserves NoTornCommit/GCSafety: it removes only
  objects owned exclusively by the GC'd version (manifests[v].objs \ stillRef),
  which no surviving or pinned manifest references; pinned versions are excluded
  by `v notin PinnedVersions`.

### 2. LIVENESS check (WriterEventuallyCommits, fair Spec, no symmetry) ###
$ java -cp tla2tools.jar tlc2.TLC -config commit_protocol_liveness.cfg \
      -workers auto commit_protocol.tla

EXPECTED: No violation. From "leased", weak fairness on
{WriteDataObjects, SwapManifestOk, SwapManifestFenced} drives the writer to
"wrote" then to "committed" or "fenced". CrashWriter and ExpireLease are
adversarial (excluded from fairness). ZombieLateWrite is also excluded from
fairness AND cannot starve the swap: it never disables SwapManifestOk/Fenced
(it only adds to dataObjects), and once the orphan is durable it becomes a
stuttering step (writerObjs[w] subseteq dataObjects => dataObjects' =
dataObjects). So no infinite non-progress cycle is introduced.  QED

### 3. NON-VACUITY PROBES (each EXPECTED to be REFUTED == reachable) ###
$ java -cp tla2tools.jar tlc2.TLC -config commit_protocol_probes.cfg \
      -workers auto commit_protocol.tla     # one INVARIANT at a time

  PROBE-A  NoRaceProbe (carried from v1) — REFUTED.
    Trace (v1, still valid): W1 AcquireLease(v1) -> W1 WriteDataObjects ->
    ExpireLease -> W2 AcquireLease(v1) -> W2 WriteDataObjects. Both staged for
    v1; the same-version race is reachable.

  PROBE-B  DistinctIdsProbe (NEW — the DA-1 non-vacuity probe) — REFUTED.
    Same trace as PROBE-A, but the refuting state now exhibits
    writerObjs[w1] = { "v1-ww1-a1-k1" } and writerObjs[w2] = { "v1-ww2-a1-k1" }
    — DISTINCT ids for the same target version v1. This is the exact collapse
    v1 hid: in v1 both were { "1-1" } and the union was idempotent. The
    refutation demonstrates the model can now REPRESENT two writers racing the
    same version with distinct ids, so NoTornCommit is tested against a genuine
    same-version race rather than satisfied vacuously.

  PROBE-C  ZombieWroteProbe (NEW — proves the late-PUT attack is exercised) —
    REFUTED. Trace: W1 AcquireLease(v1) -> W1 WriteDataObjects -> ExpireLease ->
    W2 AcquireLease(v1) -> W2 WriteDataObjects -> W2 SwapManifestOk (commits v1)
    -> W1 SwapManifestFenced. Now W1 is "fenced", its orphan objs are durable
    (writerObjs[w1] subseteq dataObjects), and v1 is committed — i.e. a fenced
    writer's stale object coexists with the live committed snapshot.
    OrphansNeverReferenced + NoOverwriteOfReferenced hold across this state
    (W1's orphan id is NOT in manifests[v1].objs = W2's id), so the late write
    is provably harmless. ZombieLateWrite additionally exercises the fenced
    writer re-PUTting AFTER fencing (lands on the same orphan key — a stutter,
    still harmless).
